2. Difference between localstorage vs sessionstorage and cookies

localstorage

localStorage is a way to store data on the client's computer. It allows the saving of key/value pairs in a web browser and it stores data with no expiration date. localStorage can only be accessed via JavaScript, and HTML5

sessionstorage

stores data only for a session, meaning that the data is stored until the browser (or tab) is closed

Cookie

Stores data that has to be sent back to the server with subsequent XHR requests. Its expiration varies based on the type and the expiration duration can be set from either server-side or client-side . Cookies are primarily for server-side reading (can also be read on client-side), localStorage and sessionStorage can only be read on client-side.

3. What is an event loop in JavaScript ?

JavaScript event loop is the mechanism that enables asynchronous programming in JavaScript. It is responsible for managing the order in which the code is executed in response to events, such as user actions, server responses, and timers.

The event loop has two main components: the call stack and the event queue. The call stack is a data structure that keeps track of the function calls in the current execution context. The event queue is a data structure that holds the events waiting to be processed.
